# Digital Forensics Autopsy Project

## Overview
This repository contains an academic digital forensics investigation project completed for educational purposes. The project demonstrates the use of Autopsy and FTK Imager to examine simulated digital evidence, review forensic artifacts, verify file integrity, and prepare formal investigative documentation.

## Disclaimer
This project is based on a simulated academic scenario. All names, identities, case details, and evidence materials are fictional or simulated for educational purposes. No real criminal evidence, real victim information, or real personal data is included. Images used in this project are benign simulation materials, including cat images used as mock evidence.

## Tools Used
- Autopsy
- FTK Imager
- HashMyFiles
- Windows forensic analysis environment
- HTML, CSV, PDF, and document-based forensic reports

## Project Objectives
- Examine simulated digital evidence using forensic tools
- Review files, metadata, and exported forensic artifacts
- Perform hash comparison and evidence integrity verification
- Organize evidence into formal exhibit folders
- Maintain chain of custody and evidence documentation
- Prepare a final forensic report based on the simulated case scenario

## Repository Structure
- `Exhibit_A_Additional_Evidences/`  
  Contains additional supporting evidence, including hash comparison evidence and camera-related simulated evidence.

- `Exhibit_B_COC_EOI_and_IRN_Forms/`  
  Contains chain of custody, evidence of inventory, and investigation-related forms.

- `Exhibit_C_Hash_Report_and_Hash_Verification/`  
  Contains hash reports and verification materials used to confirm evidence integrity.

- `Exhibit_D_Electronic_Report/`  
  Contains electronic report outputs generated during the forensic examination.

- `Exhibit_E_Forensic_Report/`  
  Contains the final forensic report for the academic investigation.

- `Final Project Question's Answers/`  
  Contains final project responses and supporting written answers.
